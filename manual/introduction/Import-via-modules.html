虽然通过脚本标签导入three.js是一种快速启动和运行的好方法，但对于大型的项目，它有一些缺点，例如:

  • 您必须手动获取并包含库的副本作为项目源代码的一部分
  • 更新库的版本是一个手动过程
  • 签入新版本的库时，版本控制差异会被构建文件的许多行混乱

通过模块导入

Three.js作为npm模块发布, see: [link:https://www.npmjs.com/package/three npm]. 这意味着你需要做的就是将three.js包含在你的项目中运行 "npm install three"

导入模块

假设您使用[link:https://webpack.github.io/ Webpack]或[link:https://github.com/substack/node-browserify Browserify]等工具打包文件, 这允许您通过捆绑所有依赖项来在浏览器中"require('modules)."

您现在应该能够将模块导入源文件并继续按常规使用它.

var THREE = require('three'); var scene = new THREE.Scene(); ...

您还可以利用[link:https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/import ES6 import syntax]

import * as THREE from 'three'; const scene = new THREE.Scene(); ...

或者如果您只想导入three.js库的选定部分, 例如Scene:

import { Scene } from 'three'; const scene = new Scene(); ...

注意事项

目前,无法以这种方式导入“examples/js“目录中的文件. 这是由于一些文件依赖于THREE的全局命名空间污染. 有关更多信息,请参阅 Transform `examples/js` to support modules #9562.
