加载3D模型


3D模型有数百种文件格式, 每个具有不同的用途,各种功能,和不同的复杂性. 虽然 three.js 提供许多 loaders, 选择正确的格式和工作流程将在以后节省时间. 有些格式难以使用, 实时体验效率低下，或者此时根本没有完全支持.

本指南提供了针对大多数用户推荐的工作流程,以及如果事情未按预期进行操作的建议.

在我们开始之前

如果您不熟悉运行本地服务器, begin with [link:#manual/introduction/How-to-run-things-locally how to run things locally] 第一, 通过正确托管文件,可以避免查看3D模型的许多常见错误.

推荐工作流

在可能的情况下,我们建议使用glTF(GL传输格式). Both .GLB and .GLTF 格式的版本得到很好的支持. 因为glTF专注于运行时资源交互, 它传输紧凑,加载速度快. 功能包括网格,材质,纹理,皮肤,骨架,变形目标,动画,灯光和相机.

公共域glTF文件可在Sketchfab等网站上获得,或者各种工具包括glTF导出

  • glTF-Blender-Exporter by the Khronos Group
  • COLLADA2GLTF by the Khronos Group
  • FBX2GLTF by Facebook
  • OBJ2GLTF by Analytical Graphics Inc
  • Substance Painter by Allegorithmic
  • Modo by Foundry
  • Toolbag by Marmoset
  • …and many more

If your preferred tools do not support glTF, consider requesting glTF export from the authors, or posting on the glTF roadmap thread.

当glTF不是一个选项时,FBX,OBJ或COLLADA等流行格式也可以定期维护.

常见问题

You've spent hours modeling an artisanal masterpiece, you load it into the webpage, and — oh no! 😭 It's distorted, miscolored, or missing entirely. Start with these troubleshooting steps:

  1. 检查JavaScript控制台是否有错误，并确保在调用.load()时记录结果时使用了onError回调
  2. 在另一个应用程序中查看模型. For glTF, drag-and-drop viewers are available for three.js and babylon.js. If the model appears correctly in one or more applications, file a bug against three.js. If the model cannot be shown in any application, we strongly encourage filing a bug with the application used to create the model.
  3. 尝试将模型向上或向下缩放1000倍。许多模型的缩放比例不同，如果相机在模型内部，则可能不会出现大型模型.
  4. 在网络选项卡中查找失败的纹理请求, like C:\\Path\To\Model\texture.jpg. 使用相对于模型的路径,例如 images/texture.jpg — 这可能需要在文本编辑器中编辑模型文件.

寻求帮助

如果您已完成上述故障排除过程，但您的模型仍无法正常工作, 寻求帮助的正确方法将使您更快地获得解决方案. 在three.js论坛上 three.js论坛 发布一个问题，并尽可能, 以您可用的任何格式包含您的模型（或具有相同问题的更简单模型）. 为其他人提供足够的信息以快速重现问题 - 最好是现场演示.
